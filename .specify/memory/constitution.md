# Grove 项目章程

## 核心原则

### I. 离线优先
所有核心功能必须在无网络连接情况下完全可用。数据本地存储为首选方案，云端同步仅为可选增强。必须明确定义离线状态下的功能边界与数据持久化策略。

**理由**: 确保用户在任何环境下都能使用核心功能，提升可靠性和隐私保护。

### II. 测试先行开发 (不可协商)
强制执行 TDD 工作流：先编写失败的测试 → 实现最小代码使测试通过 → 重构。单元测试覆盖率不低于 80%。每个公共 API 必须有合约测试验证接口稳定性。

**理由**: 高质量代码的基础保障，减少回归缺陷，文档化行为预期。

### III. 性能纪律
冷启动时间从启动到可交互状态必须小于 2 秒 (iOS 15+ 真机测试)。动画必须保持 60 FPS 恒定帧率。内存使用峰值不超过 150MB。背景任务执行时间不超过 30 秒。

**理由**: 专注工具本身不应成为性能瓶颈，流畅体验是产品核心价值。

### IV. 无障碍访问 (不可协商)
完整支持 VoiceOver 屏幕阅读器，所有交互元素必须有明确的 accessibilityLabel。支持系统动态字体 (最大字号 200%)，布局不崩坏。符合 WCAG 2.1 AA 标准。

**理由**: 专注工具应服务于所有用户，包括残障人士。这是道德义务和法律要求。

### V. 极致简洁
界面遵循"内容即界面"原则。默认仅显示必要信息，高级功能渐进式披露。单屏文本不超过 500 字。每次交互步骤不超过 3 次点击完成核心操作。

**理由**: 专注工具必须消除分心，过多功能选择本身就是干扰。

### VI. 森林美学设计
视觉语言采用柔和自然色调 (绿色系为主)。动效模拟植物生长节奏 (缓动曲线 custom ease-out)。图标简洁线条风格，避免拟物化细节。深色模式为强制选项。

**理由**: 森林主题提供心理安抚效果，研究表明自然元素降低压力水平。

## 附加约束

### 技术栈要求
- **平台**: iOS 15+ / SwiftUI + UIKit 混合 (复杂动效用 UIKit)
- **语言**: Swift 5.9+ (禁⽤ Objective-C)
- **数据存储**: CoreData + CloudKit(可选同步)
- **测试**: XCTest (单元测试)+XCUITest(UI 测试)
- **架构**: MVVM-C(Clean Architecture), 依赖注入

### 禁止事项
- ❌ 禁止网络请求作为阻塞路径
- ❌ 禁止第三方分析 SDK(隐私保护)
- ❌ 禁止广告或追踪像素
- ❌ 禁止强制账户系统

### 交付质量标准
- UI 测试覆盖关键用户旅程
- Accessibility 自动化检查集成 CI
- 性能基准测试 (冷启动/帧率) 每 PR 验证
- 二进制大小增量不超过 5MB

## 开发工作流

### 分支策略
- `main`: 始终可发布状态
- `feature/[ticket-id]-[name]`: 功能开发分支
- PR 必须关联 Issue/Ticket

### 代码审查要求
- 至少 1 人批准
- 测试全部通过
- 性能检查清单完成
- Accessibility 验证截图

### 质量门控
- ✅ 测试通过率 100%
- ✅ Lint/SwiftFormat 无错误
- ✅ Xcode static analysis 无警告
- ✅ VoiceOver 手动测试通过
- ✅ 冷启动性能报告已生成

## 治理

本章程优先级高于所有其他开发实践。任何偏离必须:

1. 在 plan.md 中记录技术违规论证
2. 团队共识批准 (至少 2 人同意)
3. 设置修复 TODO 及期限
4. 记录在复杂度跟踪表中

版本控制语义:
- MAJOR: 违反基本原则删除/重写
- MINOR: 新增原则/扩展指导范围
- PATCH: 措辞澄清/示例更新
